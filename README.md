# WebGuardian 网站安全检测工具

## 项目说明

WebGuardian 是一个基于 Python 和 Streamlit 开发的网站基础安全检测工具。用户输入目标网站地址后，工具会执行可配置的站点检测和页面级安全检查，并输出安全评分、安全等级、风险提示和原始检测数据。

项目定位为轻量的安全基线检查工具，不替代专业漏洞扫描、渗透测试或持续安全监测。

当前仅支持 `http` 和 `https` 协议；`file`、`ftp`、`ssh` 等非 HTTP(S) 协议会被直接拒绝。

## 一、主要功能

### 1. HTTPS 检测

检测最终访问地址是否使用 HTTPS。当输入地址发生跳转时，以最终响应地址为准。

### 2. SSL/TLS 证书检测

基于已建立的 HTTPS 连接检测：

- HTTPS 证书是否有效
- 证书剩余有效天数

部分网络适配层无法读取证书详细信息时，只要 HTTPS 请求已成功建立，工具仍会将证书判定为有效，但剩余天数会显示为未知。

### 3. HTTP 安全响应头检测

检查以下常见安全响应头是否存在：

- `Content-Security-Policy`
- `Strict-Transport-Security`
- `X-Frame-Options`
- `X-Content-Type-Options`
- `Referrer-Policy`
- `Permissions-Policy`

这些响应头有助于降低 XSS、点击劫持、MIME 类型嗅探和不当资源加载等风险。

### 4. TRACE 方法检测

向目标站点发送 HTTP TRACE 请求，判断 TRACE 方法是否可用。若 TRACE 被启用，可能增加跨站追踪攻击风险，通常建议关闭。

### 5. 敏感路径检测

检查常见的后台、备份、调试和接口文档路径，包括：

- `/admin`、`/admin/login`
- `/backup`、`/.env`、`/.git/HEAD`
- `/test`、`/test/login`、`/debug`
- `/api/docs`、`/swagger`、`/swagger-ui`
- `/phpinfo.php`、`/server-status`
- `/actuator`、`/console`、`/docs`

工具将 `200`、重定向、`401` 和 `403` 响应视为路径可能存在。路径存在不等同于漏洞，仍需结合访问控制和实际内容判断。

### 6. 信息泄露检测

从 `Server`、`X-Powered-By` 和 `X-AspNet-Version` 等响应头中识别高信号的版本或框架特征。

普通 `Server` 字段本身不会被直接判定为问题；只有可识别的服务器、框架及版本组合才会作为暴露信息提示。

### 7. 页面级安全检查

页面级检查从首页开始进行轻量抓取，只跟进同源 HTML 链接，并限制抓取页面数与深度，不扫描下载文件和第三方站点。

可选检查项：

- 混合内容：HTTPS 页面是否加载 `http://` 资源
- 不安全表单：表单提交地址是否使用 HTTP
- Cookie 安全属性：是否具备 `Secure`、`HttpOnly`、`SameSite`
- 重定向链：是否存在 HTTPS 降级跳转或跨源跳转
- 页面响应头：重点页面是否缺少 CSP、HSTS
- 暴露性信息：源码中的内网地址、环境标记、调试开关、Source Map 或接口调试入口

结果会显示已扫描页面数、问题数量和最高风险。未发现页面级问题时，最高风险显示为“无”。
基础信息中的 IP 结果会优先显示一个 IPv4 和一个 IPv6；当两者都存在时，完整列表会按两栏展示。

### 8. 扫描模式与进度

界面提供四种模式：

- 标准模式：启用常规站点检查
- 快速模式：跳过 TRACE、敏感路径和页面级检查
- 深度模式：启用全部基础检测和页面级检查
- 自定义模式：自行选择检测项、页面子项、最大页面数和最大深度

修改任一检测选项后会自动切换为自定义模式。输入 URL 后可按 Enter 直接开始扫描。
扫描进行中可随时点击“停止检测”中断当前任务。

进度条按当前实际启用的检测项、敏感路径数量和页面检查工作量动态计算。请求失败、非 HTML 页面或跨源跳转导致的跳过项也会完成对应进度，避免进度停滞。

当输入地址校验失败时，界面会直接提示原因，不会进入扫描流程。

## 二、评分与等级

总分按已启用的基础检测项归一化为 100 分。页面级安全检查用于暴露页面风险，当前不直接计入总评分。

| 检测项 | 最高分 | 得分条件 |
|---|---:|---|
| HTTPS | 10 | 最终地址使用 HTTPS |
| SSL/TLS | 20 | 证书有效 10 分；剩余天数大于 7 天再得 10 分 |
| HTTP 安全响应头 | 30 | 每个已配置的安全头计 5 分 |
| 信息泄露 | 20 | 未发现服务器版本特征和框架特征，各计 10 分 |
| TRACE | 10 | TRACE 方法未启用 |
| 敏感路径 | 10 | 未发现内置敏感路径 |

| 分数范围 | 等级 |
|---|---|
| 85 - 100 | A级 |
| 70 - 84 | B级 |
| 0 - 69 | C级 |

关闭某项基础检测后，该项不会计入可得分总分。

## 三、项目结构

```text
main.py                    # 直接启动 Streamlit 应用
scanner/
  options.py               # 扫描配置模型
  scanner.py               # 主扫描编排
  utils/
    control.py             # 扫描中断信号
    net.py                 # 请求会话、URL 校验和地址规范化
    scan_progress.py       # 按工作量计算扫描进度
    security_score.py      # 安全评分与等级规则
    tls.py                 # TLS/SSL 证书检测
  pagecheck/
    page_scan.py           # 同源页面抓取与检查编排
    page_checks.py         # HTML 解析及页面安全检查规则
    page_results.py        # 页面发现、风险统计和结果聚合
ui/
  app.py                   # 页面流程、URL 表单和扫描调用
  scan_config.py           # 扫描模式、配置控件和配置校验
  scan_runtime.py          # 扫描状态、后台执行、停止控制和错误展示
  scan_errors.py           # 错误归类与友好提示
  result_view.py           # 检测结果展示
```

## 四、安装与运行

### 1. Python 版本要求

建议使用 **Python 3.10 或更高版本**。项目代码使用了联合类型注解等语法，低于 3.10 的解释器不兼容。

### 2. 创建虚拟环境

Windows PowerShell：

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
```

macOS 或 Linux：

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. 安装依赖

```bash
pip install streamlit requests cryptography beautifulsoup4
```

### 4. 启动应用

在项目根目录执行：

```powershell
py main.py
```

`main.py` 已包含 Streamlit 启动逻辑，不需要再执行 `streamlit run`。启动成功后，终端会显示本地访问地址，默认通常为 `http://localhost:8501`。

## 五、使用方式

1. 选择预设扫描模式，或进入自定义模式。
2. 需要页面级检查时，启用该开关并至少选择一个页面子项。
3. 输入完整网站地址，例如 `https://example.com`。
4. 按 Enter 或点击“开始检测”。
5. 查看安全评分、基础检测结果、页面级风险、过程提示和原始 JSON 数据，必要时可在检测中停止任务。

如果输入地址无效，界面会直接提示具体原因，不会开始扫描。

若所有检测项均关闭，或页面级检查未选择任何子项，应用会在开始扫描前提示修正配置。

## 六、注意事项

1. 仅检测自己拥有或已经获得明确授权的网站。
2. 本工具不尝试绕过认证、WAF、访问控制或其他安全机制。
3. 检测结果会受到网络环境、代理、CDN、WAF、目标站点配置和证书链的影响。
4. 敏感路径检查仅使用内置的有限路径列表，路径存在不代表可被未授权访问。
5. 页面级检查受最大页数和最大深度限制，不能代表整个站点的完整审计结果。
6. TRACE、Cookie 和响应头结果可能因反向代理或中间件处理而与源站实际配置不同。
7. `0.0.0.0`、`::` 以及 `224.0.0.0-255.255.255.255` 这类地址会被当作无效输入。
